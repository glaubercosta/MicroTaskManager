# Discovery Checklist Template

Use this checklist before implementation for non-trivial work.

## Problem Framing
- [ ] Business objective is explicit and measurable
- [ ] Scope and non-goals are explicit
- [ ] Constraints are explicit (technical, compliance, timeline)
- [ ] Source-of-truth systems/docs/owners identified

## Domain Understanding
- [ ] Core entities/components mapped
- [ ] Dependencies and interfaces mapped
- [ ] Failure modes listed
- [ ] Operational context captured (runtime, deployment, data boundaries)

## Questions and Unknowns
- [ ] At least 10 critical questions listed
- [ ] High-impact unknowns prioritized
- [ ] Open questions have owners
- [ ] Decision deadline for each critical unknown defined

## Assumptions
- [ ] Critical assumptions documented
- [ ] Validation method defined for each assumption
- [ ] Validation checkpoint defined
- [ ] Fallback/mitigation defined if assumption fails

## Options and Trade-offs
- [ ] At least 2 options compared
- [ ] Trade-offs explicit (cost, complexity, latency, risk)
- [ ] Chosen approach justified
- [ ] Rejected options documented with reason

## Quality and Safety
- [ ] Acceptance criteria measurable
- [ ] Quality gates identified for this scope
- [ ] Contract impact assessed
- [ ] Observability and rollback approach defined

## Ready-to-Implement Decision
- [ ] Go: start implementation
- [ ] Hold: missing critical evidence
- [ ] Risk-accepted: proceed with explicit residual risks
